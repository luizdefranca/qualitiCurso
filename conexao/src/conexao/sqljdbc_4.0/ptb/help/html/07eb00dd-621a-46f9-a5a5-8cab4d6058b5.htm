Configurando como os valores de java.sql.Time são enviados ao servidor

Se você usar um objeto java.sql.Time object ou o tipo java.sql.Types.TIME JDBC para definir um parâmetro, você poderá configurar como o valor de java.sql.Time será enviado ao servidor: como um tipo SQL Servertime ou como um tipo datetime.

Esse cenário se aplica ao usar um dos seguintes métodos:

  • SQLServerCallableStatement.registerOutParameter(int, int)

  • SQLServerCallableStatement.registerOutParameter(int, int, int)

  • SQLServerCallableStatement.setTime

  • SQLServerPreparedStatement.setTime

  • SQLServerCallableStatement.setObject

  • SQLServerPreparedStatement.setObject

Você pode configurar como o valor de java.sql.Time será enviado usando a propriedade de conexão sendTimeAsDatetime. Para obter mais informações, consulte Definindo as propriedades de conexão.

Você pode modificar programaticamente o valor da propriedade de conexão sendTimeAsDatetime com SQLServerDataSource.setSendTimeAsDatetime.

As versões de SQL Server anteriores ao SQL Server 2008 não suportam o tipo de dados time. Dessa forma, os aplicativos que utilizam o java.sql.Time geralmente armazenam os valores de java.sql.Time como tipos de dados datetime ou smalldatetimeSQL Server.

Se quiser usar os tipos de dados datetime e smalldatetimeSQL Server ao trabalhar com valores java.sql.Time, você deverá definir a propriedade de conexão sendTimeAsDatetime para true. Se quiser usar o tipo de dados timeSQL Server ao trabalhar com valores java.sql.Time, você deverá definir a propriedade de conexão sendTimeAsDatetime para false.

Esteja ciente de que, ao enviar valores java.sql.Time para um parâmetro cujo tipo de dados também pode armazenar a data, as datas padrão serão diferentes dependendo de se o valor de java.sql.Time for enviado como um valor datetime (1/1/1970) ou time (1/1/1900). Para obter mais informações sobre as conversões de dados ao enviar dados a um SQL Server, consulte Usando dados de data e hora .

No SQL Server JDBC Driver 3.0, sendTimeAsDatetime é verdadeiro por padrão. Em uma futura versão, a propriedade de conexão sendTimeAsDatetime poderá ser definida como falso por padrão.

Para assegurar que seu aplicativo continue a funcionar como esperado, independentemente do valor padrão da propriedade de conexão sendTimeAsDatetime, você pode:

  • Usar o java.sql.Time ao trabalhar com o tipo de dados timeSQL Server.

  • Usar o java.sql.Timestamp ao trabalhar com os tipos de dados datetime, smalldatetime e datetime2SQL Server.
