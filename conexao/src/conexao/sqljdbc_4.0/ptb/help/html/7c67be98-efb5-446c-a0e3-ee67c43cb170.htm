Interface SQLXML

O driver JDBC fornece suporte para a API do JDBC 4.0, que apresenta a interface java.sql.SQLXML. A interface SQLXML define métodos para interagir com dados XML e manipulá-los. O tipo de dados SQLXML mapeia para o tipo de dados xml do SQL Server.

A interface SQLXML fornece métodos para acessar o valor de XML valor como um String, um Reader ou Writer, ou como um Stream. O valor XML também pode ser acessado através de um Source ou definido como um Result, que é usado com APIs do Analisador de XML como Modelo de Objeto de Documento (DOM), API Simples para XML (SAX) e API de Fluxo para XML (StAX), assim como transformações XSLT e XPath.

Comentários

A tabela a seguir descreve os métodos definidos na interface SQLXML:

Sintaxe de método

Descrição de método

void free()

Esse método libera o objeto SQLXML e os recursos que ele contém.

InputStream getBinaryStream()

Retorna um fluxo de entrada para ler dados do SQLXML.

Reader getCharacterStream()

Retorna os dados de XML como um objeto java.io.Reader ou como um fluxo de caracteres.

T extends Source T getSource(Class<T> sourceClass)

Retorna um Source para ler o valor XML especificado por esse objeto SQLXML.

O método getSource dá suporte às seguintes origens: javax.xml.transform.dom.DOMSource, javax.xml.transform.sax.SAXSource, javax.xml.transform.stax.StAXSource e java.io.InputStream.

String getString()

Retorna uma representação de cadeia de caracteres do valor XML designado por este objeto SQLXML.

OutputStream setBinaryStream()

Recupera um fluxo que pode ser usado para gravar o valor XML que este objeto SQLXML representa.

Writer setCharacterStream()

Retorna um fluxo para ser usado para gravar o valor XML que este objeto SQLXML representa.

T extends Result T setResult(Class<T> resultClass)

Retorna um Result para definir o valor XML especificado por esse objeto SQLXML.

O método setResult dá suporte às seguintes origens: javax.xml.transform.dom.DOMResult, javax.xml.transform.sax.SAXResult, javax.xml.transform.stax.StaxResult e java.io.OutputStream.

void setString(String value)

Define um valor XML designado por esse objeto SQLXML para a representação String especificada.

Os aplicativos podem ler e gravar valores XML para ou de um objeto SQLXML apenas uma vez.

Quando o método free() é chamado, um objeto SQLXML se torna inválido e não é nem legível nem gravável. Se o aplicativo tentar invocar um método naquele objeto SQLXML diferente do método free(), uma exceção será lançada.

O objeto SQLXML não fica legível nem gravável quando o aplicativo chama qualquer um dos métodos getter a seguir: getSource, getCharacterStream, getBinaryStream e getString.

O objeto SQLXML não fica legível nem gravável quando o aplicativo chama qualquer um dos métodos setter a seguir: setResult, setCharacterStream, setBinaryStream e setString.
