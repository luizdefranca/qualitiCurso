Trabalhando com tipos de dados (JDBC)

A principal função do Microsoft JDBC Driver para SQL Server é permitir que os desenvolvedores Java acessem dados contidos nos bancos de dados do SQL Server. Para realizar isso, o driver JDBC atua como mediador da conversão entre tipos de dados do SQL Server e tipos e objetos da linguagem Java.

Para acompanhar uma discussão detalhada dos tipos de dados do SQL Server e do driver JDBC, incluindo as suas diferenças e como eles são convertidos para tipos de dados da linguagem Java, consulte Entendendo os tipos de dados do JDBC Driver.

Para trabalhar com tipos de dados do SQL Server, o driver JDBC fornece métodos get<Type> e set<Type> para as classes SQLServerPreparedStatement e SQLServerCallableStatement, e fornece métodos get<Type> e update<Type> para a classe SQLServerResultSet. Qual método você usa depende do tipo de dados com os quais você está trabalhando, e se você está usando conjuntos de resultados ou consultas.

Os tópicos nesta seção descrevem como usar os tipos de dados do driver JDBC para acessar dados do SQL Server em seus aplicativos Java.
