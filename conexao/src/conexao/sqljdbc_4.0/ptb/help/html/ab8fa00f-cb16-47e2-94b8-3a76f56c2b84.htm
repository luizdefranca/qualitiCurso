Entendendo diferenças de tipo de dados

Há várias diferenças entre os tipos de dados da linguagem de programação Java e os tipos de dados do SQL Server. O Microsoft JDBC Driver para SQL Server ajuda a facilitar essas diferenças através de vários tipos de conversões.

Tipos de caractere

Os tipos de dados de cadeia de caracteres do JDBC são CHAR, VARCHAR e LONGVARCHAR. O driver JDBC oferece suporte à API do JDBC 4.0. No JDBC 4.0, os tipos de dados de cadeia de caracteres do JDBC podem ser também NCHAR, NVARCHAR e LONGNVARCHAR. Estes novos tipos de cadeia de caracteres mantêm tipos de caracteres nativos de Java em formato Unicode e remove a necessidade de executar alguma conversão ANSI-para-Unicode ou Unicode-para-ANSI.

Tipo

Descrição

Comprimento fixo

Os tipos de dados char e nchar do SQL Server mapeiam diretamente para os tipos CHAR e NCHAR do JDBC. Estes são tipos de comprimentos fixos com preenchimento fornecidos pelo servidor no caso em que a coluna possui SET ANSI_PADDING ON. O preenchimento é sempre ligado para nchar, mas para char, no caso em que as colunas de char de servidor não são preenchidas, o driver JDBC adiciona o preenchimento.

Comprimento variável

Os tipos varchar e nvarchar do SQL Server mapeiam diretamente para os tipos VARCHAR e NVARCHAR do JDBC, respectivamente.

Long

Os tipos SQL Server text e ntext são mapeados para o LONGVARCHAR e LONGNVARCHAR do JDBC, respectivamente. São tipos que foram substituídos desde o SQL Server 2005, de modo que no lugar você deve usar tipos de valores grandes, varchar(max) ou nvarchar(max).

Usar os métodos update<Numeric Type> e updateObject (int, java.lang.Object) não funcionará em colunas de servidor de text e ntext. Porém, usar o método setObject com um tipo de conversão de caractere especificado tem suporte para colunas de servidor de text e ntext.

Tipos de cadeia de caracteres binária

Os tipos de cadeia de caracteres binários do JDBC são BINARY, VARBINARY e LONGVARBINARY.

Tipo

Descrição

Comprimento fixo

O tipo de dados binary do SQL Server mapeia diretamente para o tipo BINARY do JDBC. Esse é um tipo de comprimento fixo com preenchimento fornecido pelo servidor no caso em que a coluna possui SET ANSI_PADDING ON. Quando as colunas de char de servidor não são preenchidas, o driver JDBC adiciona o preenchimento.

O tipo timestamp do SQL Server é um tipo BINARY do JDBC com o comprimento fixo de 8 bytes.

Comprimento variável

O tipo de dados varbinary do SQL Server mapeia para o tipo VARBINARY do JDBC.

O tipo udt no SQL Server é mapeado para o JDBC como um tipo VARBINARY.

Long

O tipo image do SQL Server é mapeado para o tipo LONGVARBINARY do JDBC. Esse tipo foi substituído desde o SQL Server 2005, de modo que no lugar você deve usar um tipo de valor grande, varbinary(max).

Tipos numéricos exatos

Os tipos numéricos exatos do JDBC mapeiam diretamente para os tipos do SQL Server correspondentes.

Tipo

Descrição

BIT

O tipo BIT do JDBC representa um único bit que pode ser 0 ou 1. Ele mapeia para um tipo bit do SQL Server.

TINYINT

O tipo TINYINT do JDBC representa um único byte. Ele mapeia para um tipo tinyint do SQL Server.

SMALLINT

O tipo SMALLINT do JDBC representa um inteiro de 16 bits com sinal. Ele mapeia para um tipo smallint do SQL Server.

INTEGER

O tipo INTEGER do JDBC representa um inteiro de 32 bits com sinal. Ele mapeia para um tipo int do SQL Server.

BIGINT

O tipo BIGINT do JDBC representa um inteiro de 64 bits com sinal. Ele mapeia para um tipo bigint do SQL Server.

NUMERIC

O tipo NUMERIC do JDBC representa um valor decimal de precisão fixa que contém valores de precisão idêntica. O tipo NUMERIC mapeia para o tipo numeric do SQL Server.

DECIMAL

O tipo DECIMAL do JDBC representa um valor decimal de precisão fixa que contém valores da precisão especificada pelo menos. O tipo DECIMAL mapeia para o tipo decimal do SQL Server.

O tipo DECIMAL do JDBC também mapeia para os tipos money e smallmoney do SQL Server, que são tipos decimais de precisão fixa específicos e que são armazenados em 8 ou 4 bytes, respectivamente.

Tipos numéricos aproximados

Os tipos numéricos aproximados do JDBC são REAL, DOUBLE e FLOAT.

Tipo

Descrição

REAL

O tipo REAL do JDBC tem sete dígitos de precisão (precisão única) e mapeia diretamente para o tipo real do SQL Server.

DOUBLE

O tipo DOUBLE do JDBC tem 15 dígitos de precisão (precisão dupla) e mapeia diretamente para o tipo float do SQL Server. O tipo FLOAT do JDBC é um sinônimo do DOUBLE. Como pode haver confusão entre FLOAT e DOUBLE, o DOUBLE é preferido.

Tipos de Data e Hora

O tipo TIMESTAMP do JDBC mapeia para os tipos datetime e smalldatetime do SQL Server. O tipo datetime é armazenado em dois inteiros de 4 bytes. O tipo smalldatetime contém as mesmas informações (data e hora), mas com menos exatidão, em dois inteiros pequenos de 2 bytes.

O tipo timestamp do SQL Server é um tipo de cadeia de caracteres binária de comprimento fixo. Ele não mapeia para nenhum dos tipos de hora de JDBC: DATE, TIME ou TIMESTAMP.

Mapeamento de tipo personalizado

O recurso de mapeamento de tipo personalizado do JDBC que usa as interfaces do SQLData para os tipos avançados do JDBC (UDTs, Struct, e assim por diante). não é implementado no driver JDBC.
