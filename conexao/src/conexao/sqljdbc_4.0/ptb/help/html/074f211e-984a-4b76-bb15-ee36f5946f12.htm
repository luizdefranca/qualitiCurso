Novidades do JDBC Driver

O Microsoft JDBC Driver 4.0 para SQL Server inclui vários novos recursos e aprimoramentos.

Informações sobre como conectar a um banco de dados do SQL Azure

Agora, há um tópico com informações sobre como conectar a um banco de dados do SQL Azure. Consulte Conectando a um banco de dados do SQL Azure para obter mais informações.

Suporte para alta disponibilidade e recuperação de desastres

O Microsoft JDBC Driver 4.0 para SQL Server oferece alta disponibilidade e conexões de recuperação de desastres aos grupos de disponibilidade AlwaysOn no SQL Server 2012. Consulte Suporte a driver JDBC para alta disponibilidade e recuperação de desastre para obter mais informações.

Usando a autenticação integrada Kerberos para conectar-se ao SQL Server

O Microsoft JDBC Driver 4.0 para SQL Server tem quatro tipos de autenticação integrada Kerberos para aplicativos que se conectam a bancos de dados do SQL Server. Para obter mais informações, consulte Usando a autenticação integrada Kerberos para conectar-se ao SQL Server. (A autenticação integrada Kerberos Tipo 2 está disponível nas versões do Microsoft JDBC Driver para SQL Server anteriores à versão 4.0.)

Acessando informações de diagnóstico no log de eventos estendidos

Você pode acessar informações no log de eventos estendidos do servidor para entender as falhas de conexão. Para obter mais informações, consulte Acessando informações de diagnóstico no log de eventos estendidos.

Suporte adicional para colunas esparsas

Se o seu aplicativo já acessa dados em uma tabela que usa colunas esparsas, você deverá ver uma melhora no desempenho. Você pode obter informações sobre colunas (incluindo informações sobre colunas esparsas) com Método getColumns (SQLServerDatabaseMetaData). Para obter mais informações sobre colunas esparsas do SQL Server, consulte Usando colunas esparsas.

O método isSparseColumnSet é adicionado à classe SQLServerResultSetMetaData para indicar se uma coluna de um conjunto de resultados é um conjunto de colunas esparsas.

Xid.getFormatId

Antes do Microsoft JDBC Driver 4.0 para SQL Server, o driver JDBC recebia um identificador de formato do aplicativo, mas não passava o valor ao servidor de banco de dados. Em vez disso, o driver JDBC passava um identificador de formato de 0 ao servidor de banco de dados.

Com o Microsoft JDBC Driver 4.0 para SQL Server, o driver JDBC passara o identificador de formato do aplicativo para o servidor de banco de dados. Para obter o comportamento atualizado, verifique se o arquivo sqljdbc_xa.dll no servidor foi atualizado. Para obter mais informações sobre a cópia de uma versão atualizada do sqljdbc_xa.dll para o servidor, consulte Compreendendo transações XA.

O Itanium não tem suporte em aplicativos do JDBC Driver 4.0

Os aplicativos do Microsoft JDBC Driver 4.0 para SQL Server não têm suporte para execução em computadores com Itanium.
