Requisitos de sistema para o JDBC Driver

Para acessar dados de um banco de dados do SQL Server usando o Microsoft JDBC Driver para SQL Server, os seguintes componentes devem estar instalados no computador:

  • Microsoft JDBC Driver para SQL Server

  • Java Runtime Environment

Requisitos do Java Runtime Environment

A partir do Microsoft JDBC Driver para SQL Server, o suporte do driver JDBC para a API da Especificação do JDBC (Java Database Connectivity) foi estendido para incluir a API do JDBC 4.0. O JDBC 4.0 foi introduzido como parte do Sun Java SE Development Kit (JDK) 6.0 e do Java Runtime Environment (JRE) 6.0. O JDBC 4.0 é um superconjunto do JDBC 3.0.

O Microsoft JDBC Driver para SQL Server oferece suporte ao JDBC 3.0 e ao JDBC 4.0.

Ao implantar o Microsoft JDBC Driver para SQL Server nos sistemas operacionais Windows e UNIX, você deve usar os pacotes de instalação sqljdbc_<version>_enu.exe e sqljdbc_<version>_enu.tar.gz, respectivamente. Para obter mais informações sobre como implantar o JDBC Driver, consulte o tópico Implantando o JDBC Driver.

Para oferecer suporte à compatibilidade com versões anteriores e a possíveis cenários de atualização, o JDBC Driver inclui duas bibliotecas de classes de JAR em cada pacote de instalação: sqljdbc.jar e sqljdbc4.jar.

JAR

Descrição

sqljdbc.jar

A biblioteca de classes sqljdbc.jar dá suporte para o JDBC 3.0.

A biblioteca de classes sqljdbc.jar requer um JRE (Java Runtime Environment) de versão 5.0. O uso de sqljdbc.jar no JRE 6.0 lançará uma exceção durante a conexão com um banco de dados.

O JDBC Driver não oferece suporte ao JRE 1.4. Você deve atualizar o JRE 1.4 para JRE 5.0 ou JRE 6.0 ao usar o driver JDBC. Em alguns casos, poderá ser necessário recompilar o aplicativo porque talvez ele não seja compatível com o JDK 5.0 ou posterior. Para obter mais informações, consulte a documentação no site da Sun Microsystems.

sqljdbc4.jar

A biblioteca de classes sqljdbc4.jar dá suporte para o JDBC 4.0. Ela inclui todos os recursos de sqljdbc.jar e também os novos métodos do JDBC 4.0.

A biblioteca de classes sqljdbc4.jar requer um JRE (Java Runtime Environment) de versão 6.0. O uso de sqljdbc4.jar no JRE 1.4 ou 5.0 lançará uma exceção.

Use sqljdbc4.jar quando seu aplicativo tiver que ser executado no JRE 6.0, mesmo que ele não use recursos do JDBC 4.0.

Observe que o driver JDBC foi criado para funcionar com, e ter suporte de, todas as principais máquinas virtuais Java equivalentes à Sun, mas é testado no Sun JRE 5.0 ou posterior.

Requisitos do SQL Server

O driver JDBC oferece suporte a conexões ao SQL Server 2005 e a versões posteriores.

Requisitos do sistema operacional

O driver JDBC foi criado para operar em qualquer sistema operacional que ofereça suporte ao uso de uma JVM (Máquina Virtual Java). Entretanto, apenas os sistemas operacionais Sun Solaris, SUSE Linux, Windows Vista e Windows 7 foram testados oficialmente.

Idiomas com suporte

O driver JDBC dá suporte a todos os agrupamentos de coluna do SQL Server. Para obter mais informações sobre os agrupamentos que têm suporte no driver JDBC, consulte Recursos internacionais do JDBC Driver.

Para obter mais informações sobre agrupamentos, consulte "Trabalhando com agrupamentos" nos Manuais Online do SQL Server.
